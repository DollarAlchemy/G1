HTML Basics

"HTML does not make things look good. It makes things exist. That is its only job, and it does it well." — Systemcraft, First Lesson

HTML stands for HyperText Markup Language. It is the foundation of every page on the web. Every heading, paragraph, image, link, and button you see in a browser exists because an HTML tag told the browser it should be there.

What is a Tag?

HTML is written using tags. A tag is a word wrapped in angle brackets. Most tags come in pairs: an opening tag and a closing tag. The closing tag has a forward slash. 

<p>This is a paragraph.</p>

<h1>This is a big heading.</h1>

<h2>This is a smaller heading.</h2>

<a href="other-page.html">This is a link.</a>

The Structure of Every HTML Page

Every HTML file follows the same basic skeleton. This is not optional — it's the contract the browser expects. 

<!DOCTYPE html>          <!-- tells browser: this is HTML5 -->
<html lang="en">          <!-- the root element wrapping everything -->
  <head>                  <!-- invisible settings: title, fonts, CSS -->
    <meta charset="UTF-8">
    <title>My Page</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>                  <!-- everything visible goes here -->
    <h1>Hello, world.</h1>
    <p>This is my first page.</p>
  </body>
</html>

Essential Tags to Know

You don't need to memorize all of HTML. These ten tags will build most of what you need:

<h1> to <h6>     — Headings. h1 is biggest, h6 is smallest.
<p>              — Paragraph of text.
<a href="">      — A link. Put the URL inside the href="".
<img src="" alt=""> — An image. src= is the file path. alt= is description text.
<div>            — A generic container. Used for layout grouping.
<span>           — Inline container. Used to style a word inside a sentence.
<ul> / <ol>     — Unordered (bulleted) or ordered (numbered) list.
<li>             — A list item. Goes inside <ul> or <ol>.
<button>         — A clickable button.
<nav>, <main>, <footer>, <header> — Structural sections (semantic HTML).
✦ ✦ ✦

Practice Exercise

Open VS Code. Create a new file called practice.html. Type the skeleton above by hand — not copy-paste. Then add:

  • An <h1> with your name
  • A <p> describing this project
  • A <ul> with 3 list items
  • A <a> linking to any URL

Open the file in your browser (drag it into the browser window, or use Live Server in VS Code). See what appears. That is your first page.

The lesson is not in reading this. It is in typing, breaking, fixing, and reading the error. Do not skip the exercise.
